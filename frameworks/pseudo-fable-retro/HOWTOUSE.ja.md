# pseudo-fable-retro — 使い方

[English](HOWTOUSE.md) | 日本語

導入後の日常運用ガイド。設計思想と導入手順は [README.ja.md](README.ja.md) を参照。retro はどの構成にも 2 つの儀式を足す:セッションの起動/終了シーケンス(`session-bootstrap`)と、教訓収穫ループ(`retro`)。

## 導入すると何が変わるか

- 以前の作業を継続するセッションが**記憶ではなくファイルから起動**する:状態ファイル → 委任台帳 → チケット → `git log`/`git status`、次にドリフト確認、そして作業前に「現在地」の 1 メッセージ復唱。
- 終了するセッションが**意図的にチェックポイントを残す**:状態ファイル更新、作業ツリーの明示的な退避(読めるメッセージ付き commit/stash)、3 行ハンドオフ(今日やったこと/次のアクション/未解決リスク)。
- マイルストーン後 — およびバウンス・リバート・長引いた作業の後 — に、実際に起きた失敗から**最大 2 ルール**を収穫し、最も狭い有効な置き場所にルーティングする。
- 定期的に、発火しないルールが**剪定**される。家訓:ルールは再発した失敗から育て、発火しなくなったら消す。

## あなたの役割

| 場面 | やること |
|---|---|
| 継続セッションの開始 | 「X の続きをやろう」と言い、OPEN シーケンスに任せる。コードに触る**前に**復唱(「タスクは X/Y、次は Z、ここがドリフト」)が来るはず — 訂正はそこで。 |
| セッションの終了 | 「今日はここまで」「セッションを閉じて」。状態ファイル更新、作業ツリーの明示的退避(黙ってダーティなまま放置しない)、3 行ハンドオフを確認。 |
| マイルストーン後・荒れた後 | 自動で走らなければ「retro して」。提案は 2 ルール以下、各ルールにトリガー・アクション・防ぐ失敗が付くはず。 |
| ルールの行き先がストアのテンプレートのとき | 配置表の定め:汎用的な教訓は**あなたに提案**され、共有ストアに黙って書き込まれることはない。承認・行き先変更・却下を判断する。 |
| 週次 / 10 タスクごと | 剪定パスを頼む:「ルール棚卸し — 発火してないのはどれ?」。削除は追加と同価値。 |

## OPEN シーケンス(見えるはずのもの)

1. 存在するものを順に読む:`.claude/state/<task-slug>.md` → `.claude/state/delegations.md` → `docs/plan/<slug>/03-tickets.md` → `git log --oneline -15` + `git status`。
2. ドリフト確認:重要事実の上位 3 件を実コードに照合。状態ファイルにない dirty な `git status` は調査。飛行中の委任は台帳を突合。
3. 短い 1 メッセージで復唱し、`Next action` から継続 — **計画済みのものを再計画しない**(完成した計画の練り直しがセッションの最初の 1 時間を溶かす)。

コンパクション後のコンテキストは「変装した新セッション」:OPEN はコンパクション後にも適用される。

## retro ループ(良い retro の形)

1. **収穫** — 失敗(バウンス、リバート、ゲートでの検出)、抵抗(2 回以上かかった作業、既知の罠の再発見)、驚き(値付けされていなかったリスク)。
2. **欠けていた一文** — 各項目に「どの文書のどの一文があれば防げたか?」。文にできない → まだ失敗を理解できていない。
3. **ルーティング**(配置表)— このリポジトリの罠 → Gotchas / ワーカーが踏む罠 → 常設ブリーフ Gotchas / プロセス失敗 → プロジェクト CLAUDE.md(トリガー付き)/ 汎用 → ストアへの提案としてあなたへ / 好み → エージェントメモリ / 一回きり → **どこにも書かない**(すべての教訓がルールに値するわけではない)。
4. **制約** — 1 retro につき最大 2 ルール。各ルールはトリガー+アクション+防ぐ失敗をセットで持つ。

## 操縦フレーズ

- 「どこまでやってたっけ?」 — 記憶で推測し始めたら、ファイルからの起動をトリガーする。
- 「セッションを閉じて」 — ツリー退避とハンドオフを含むフル CLOSE。
- 「さっきのバウンス、マイクロ retro して」 — 2 分以内・2 ルール以下。
- 「ルール棚卸しして」 — §4 の剪定パス。

## 見るべき成果物

- `.claude/state/*.md` — OPEN が起動元にし、CLOSE が更新するもの。`Next action` の 1 行が、セッションが残す最も価値ある遺産。
- プロジェクト CLAUDE.md / AGENTS.md の Gotchas — 収穫されたルールの着地点。肥大化を監視する(そのための剪定パス)。

## 想定どおり動かないとき

- **セッション開始でコードベース全読を始める** → 「状態ファイルから起動して — Learned facts は recon を安くするためにある」。
- **計画がチャットにしかないまま終了する** → 「チャットはハンドオフではない — 状態ファイルを更新してツリーを退避して」。
- **失敗ではなく思いつきでルールを足す** → 家訓を引用。すべてのルールは防ぐ失敗を名指しできなければならない。
- **CLAUDE.md が肥え続ける** → 剪定を予定に入れる。追加しかしない retro は、それ自体が壊れている(§5)。
