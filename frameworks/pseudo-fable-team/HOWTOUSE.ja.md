# pseudo-fable-team — 使い方

[English](HOWTOUSE.md) | 日本語

導入後の日常運用ガイド。設計思想と導入手順は [README.ja.md](README.ja.md) を参照。リポジトリ直下の AGENTS.md 1 枚が、リード・ワーカーを問わず全エージェントを統治する — 各自が §1–§2、自分のロールの節、共有フォーマットだけを読む。

## 導入すると何が変わるか

このファイルを読んだエージェントは、まず自分の**ロール**を解決し、そのロールの規律だけに従う:

| シグナル | ロール |
|---|---|
| プロンプトに `# TASK <id>` ブリーフがある | Worker(§4) |
| ユーザーから生の要求を受け、他エージェントを起動・指示できる | PL(§3) |
| どちらでもない(ソロセッション) | Worker 規律で動く。あなたの依頼が Contract、あなたが PL |
| 指示に明示の `Role:` 行がある | 上記すべてに優先 |

標準アサインは Opus 級セッション = PL、Sonnet / Codex = Worker。ただし判定は上の機能ルールで行われるため、同じファイルがどの席でも無改造で機能する。

## PL セッションの運用(普段あなたが話す相手)

Opus 級セッションにはテックリードに話すように接する:

- ゴールを渡すと、番号付きのテスト可能な契約に変換し、質問は推奨デフォルト付きの 1 バッチだけ(それ以外は前提として記録)。
- インターフェースに沿って分割し、タスクごとに §5 形式のフルブリーフで委任 — ブリーフなしに委任なし。並列ファンアウト前に共有インターフェースを凍結。
- 返却ごとに独立検証してから統合:ACCEPT / PATCH / BOUNCE(最大 2 回・証拠ベース)/ RECLAIM。統合(マージ後ビルド+全テスト)は PL 自身の仕事で、委任しない。
- 台帳は `.claude/state/delegations.md`。「台帳を見せて」でタスク横断のダッシュボードが見られる。

## ワーカーの運用

- **Claude サブエージェント**:ブリーフをプロンプトとして受け取り、CLAUDE.md の `@AGENTS.md` ブリッジ(またはネイティブ読込)で AGENTS.md を読む。ワーカーごとの追加設定は不要。
- **Codex**:非対話実行(`codex exec "<brief>"` — 正確な形式はローカル CLI で確認)。ブリーフと AGENTS.md がすべてを運ぶ。他には何も共有されない。
- 変則的な構成でロールを固定したいときは、指示に `Role: worker`(または `Role: PL`)の行を入れる。

## 読み方を覚えるべき 2 つのフォーマット

PL↔Worker のやり取りはすべて §5 の形なので、どの往復も一目で監査できる:

- **ブリーフ**:`# TASK <id>` + Context / Pointers / Contract / Non-goals / Constraints / Gotchas / Done means / Escalation / Report back。
- **レポート**:`RESULT: DONE | BLOCKED | DEVIATED` + Changes / Evidence / Deviations & discoveries / Not done — 40 行上限。`DONE` が正当なのは全契約項目に実出力の証拠が貼られているときだけ。スタックレポート付きの正直な `BLOCKED` は失敗ではなく成果物。

## 操縦フレーズ

- 「台帳を見せて」 — 委任ダッシュボード。
- 「そのレポート、証拠がないからバウンスして」 — ワーカーは実際のコマンド出力を貼る義務がある。
- 「ファンアウトの前にインターフェースを凍結して」
- 「Role: worker」(プロンプト内)— 単発セッションにワーカー規律を強制。

## 想定どおり動かないとき

- **エージェントが違うロールで動く** → プロンプトに実際何が入っていたか確認。ディスパッチは機械的(`# TASK` ⇒ Worker)。迷ったら明示の `Role:` 行を足す。
- **Claude エージェントがファイルを読んでいる気配がない** → CLAUDE.md 冒頭付近の `@AGENTS.md` 行を確認。またはお使いの Claude Code が AGENTS.md をネイティブに読むか実際に試す(思い込みで判断しない)。
- **AGENTS.md と CLAUDE.md でコマンドが食い違う** → Project specifics の同期漏れ。CLAUDE.md が正、そちらからコピーする。
- **より深いプロトコルが欲しくなってきた**(仮説ジャーナル、フェーズゲート、レトロ儀式)→ このファイルは蒸留された「定数」。フルファミリーに卒業する:深さは lift + orchestrate(+ blueprint)で置き換えるか、それらのスキルを併設する — その場合このシートが定数、スキルが深いプロトコルという分担になる。
